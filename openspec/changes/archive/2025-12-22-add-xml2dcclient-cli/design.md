## Context
現在は scripts 配下の個別スクリプトを順に実行して変換を進めており、実行順序の理解や中間ファイルの管理が必要になっている。
今回の変更では、XML入力からAPIクライアント生成までを1コマンドで実行できるようにする。

## Goals / Non-Goals
- Goals:
  - `xml2dcclient <xml>` で変換パイプラインを一括実行できること
  - 中間生成物をデフォルトでファイル出力しないこと
  - 既存の変換ロジックを再利用し、最小の変更で統合すること
- Non-Goals:
  - 追加の出力形式や新しいAPIクライアント生成方式の導入
  - デバッグ用の中間生成物出力フラグの実装（将来検討）

## Decisions
- Decision: CLIは commander を利用し、`xml2dcclient` コマンド名で提供する
- Decision: 変換パイプラインはメモリ上のデータ構造を受け渡す
- Decision: 生成されたクライアントの出力先は既存の `src/client/` に固定する
- Alternatives considered: 既存スクリプトをそのまま順番に呼び出す方式
  - 理由: 中間ファイル生成が避けられず、要求に合致しないため不採用

## Risks / Trade-offs
- 一括実行により失敗時の原因切り分けが難しくなる可能性がある
  - 対策: 変換ステップのエラーを分かりやすくCLIで通知する

## Migration Plan
1. CLI用の統合パイプライン関数を追加し、既存の変換ロジックを呼び出す
2. scripts の役割を整理し、CLIからの利用に統合する
3. ドキュメントを更新し、`xml2dcclient` 実行方法を明記する
